## WHAT EACH SYSTEM DOES

### Hybrid Recommendation Engine
```
INPUT: Book description + title
OUTPUT: Top 5 book recommendations

PROCESS:
1. Collaborative Filtering (60%)
   Uses SVD embeddings to find books rated similarly by users
   Best for: Popular/well-rated books, discovering patterns

2. Content-Based Filtering (25%)
   Uses TF-IDF to find semantically similar books
   Best for: Niche books, discovering by topic

3. Genre-Based Fallback (10%)
   Keyword matching when other methods uncertain
   Best for: Specific genre requests

4. Popularity Weighting (5%)
   Boost highly-rated and frequently-rated books
   Best for: Quality assurance
```

### OCR Pipeline
```
INPUT: Book cover image
OUTPUT: Extracted text candidates

PROCESS:
1. YOLO Detection
   Detects text regions in image

2. OCR Extraction
   EasyOCR → Tesseract → Preprocessing
   Returns best text found

3. Text Optimization
   Deduplication, filtering, ranking

OUTPUT: Top candidates ["Title", "Author", "Publisher", ...]
```

### Backend API
```
INPUT: Image or Query
OUTPUT: Full book details + recommendations

PROCESS:
1. Image handling
   - Save temporarily
   - Pass to CV pipeline
   - Extract OCR text
   - Query options

2. Book identification
   - Use Google Books API
   - Multi-query fallback
   - Best match scoring

3. Insights generation
   - Summary extraction
   - Tag generation
   - Score calculation
   - Hybrid recommendations

4. Response formatting
   - JSON with all data
   - Hybrid recommendations
   - Supplementary CSV recs
```
