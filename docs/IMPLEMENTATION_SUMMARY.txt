## KEY IMPROVEMENTS

1. COLLABORATIVE FILTERING
   - Uses 77,805 unique users' rating patterns
   - Finds books similar to query based on how users rated them
   - Prioritizes highly-rated books

2. CONTENT-BASED MATCHING
   - TF-IDF vectorization of book metadata
   - Finds semantically similar books
   - Falls back when no rating data available

3. POPULARITY WEIGHTING
   - Books weighted by: (avg_rating * 0.7) + (log(rating_count) * 0.3)
   - Highly-rated AND frequently-rated books preferred
   - Avoids recommending obscure/low-quality books

4. GRACEFUL DEGRADATION
   - Tries collaborative filtering first
   - Falls back to content-based if needed
   - Uses genre keywords as final fallback
   - Always returns best available recommendations

## MODEL ARCHITECTURE

Collaborative Filtering:
- Algorithm: SVD (Singular Value Decomposition)
- Factors: 50 latent dimensions
- Sparsity: 0.00003% (extremely sparse)
- Explained Variance: ~45% from 50 factors
- Similarity: Cosine distance on item factors

Content-Based Filtering:
- Algorithm: TF-IDF + Nearest Neighbors (KNN)
- Features: 10,000 max from book titles/authors
- Metric: Cosine distance
- Neighbors: 10 nearest books

Hybrid Scoring:
- Collaborative: 60% (prioritized)
- Content-Based: 25%
- Genre-Based: 10% (fallback)
- Popularity: 5% (bonus)

## USAGE EXAMPLE

```python
from backend.utils import recommend_books

# Single description
description = "A mystery novel with detective work and crime solving"
recommendations = recommend_books(description, title="Murder Mystery")

for rec in recommendations:
    print(f"{rec['title']} by {rec['author']}")
    print(f"Score: {rec['similarity_score']}, Rating: {rec['rating']}")
    print(f"Source: {rec['recommendation_source']}")
```

Output:
- Title, Author
- Similarity Score (0-1)
- Average Rating (0-10)  
- Source (collaborative/content-based/genre-based)
